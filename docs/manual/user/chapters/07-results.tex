\chapter{结果解读}

评估跑完之后，\file{output/<run>/} 下会有大量文件。本章解释每个目录里有什么、各个 metric 是什么意思、scores 是怎么归一化的，以及如何用 Python 直接读取结果做二次分析。

\section{输出目录布局}

\paragraph{多 ref / 多 sim 时的命名规则}

文件名里 \texttt{\_ref\_<R>\_sim\_<S>} 这种三段式不是冗余，是为了让 (var × sim × ref) 笛卡尔积下每对都有独立产物，互不覆盖：

\begin{itemize}
  \item 一个变量 3 sim × 2 ref → \file{metrics/} 下每 metric 6 个 NC（不会因为 sim 名相同而冲突）
  \item Comparison 阶段里所有 (sim, ref) 对会出现在同一份 \file{Portrait\_Plot\_seasonal.csv} 里，作为不同行（\texttt{Item} / \texttt{Reference} / \texttt{Simulation} 三列）
  \item Comparison 的可视化通常按"\textbf{每个 ref 一张图}"或"\textbf{所有 ref 拼在一张图}"两种风格出（具体看图种实现，见第 6 章 \cli{run} 阶段流程）
\end{itemize}

\textbf{首次熟悉时的查看顺序}：\file{report.html} → \file{comparisons/Portrait\_Plot\_seasonal/} → \file{figures/} → \file{metrics/<var>\_ref\_<R>\_sim\_<S>\_<metric>.nc}。

\section{Metrics CSV 解读}

实际产物分两类，按 ref/sim 数据形态决定：

\textbf{Grid×grid}：每 (var, ref, sim, metric) 一个 NetCDF，逐格点存值。

\file{metrics/<var>\_ref\_<R>\_sim\_<S>\_<metric>.nc} 内含一个名为 \texttt{<metric>} 的 \texttt{xarray.DataArray}，维度 \texttt{(time, lat, lon)} 或 \texttt{(lat, lon)}（climatology 模式聚合掉时间）。可用 \texttt{xarray.open_dataset()} 直接读，做空间分布分析。

\textbf{任一边 stn}：每 (var, ref, sim) 一个聚合 CSV，一行一站。

\file{metrics/<var>\_stn\_<R>\_<S>\_evaluations.csv}：

\begin{minted}{text}
ID,sim_lat,sim_lon,ref_lon,ref_lat,use_syear,use_eyear,bias,RMSE,ubRMSE,correlation,KGE,...
6101120,49.9,7.5,49.9,7.5,2000,2010,0.12,0.45,0.32,0.78,0.71,...
6101200,50.2,7.8,50.2,7.8,2000,2010,0.08,0.39,0.28,0.81,0.74,...
\end{minted}

每 metric 各占一列；前若干列是站点元信息（ID、坐标、有效年份）。

\subsection{常用 metric 含义速查}

\begin{longtable}{l p{3cm} p{8cm}}
  \toprule
  Metric & 范围 & 含义 \\
  \midrule
  \endhead
  \texttt{bias} & (-∞, +∞) & 模型 - 观测的均值偏差。0 = 无偏。同号正负向偏 \\
  \texttt{percent\_bias} & (-∞, +∞) \% & bias / 观测均值；相对偏差，方便跨变量比较 \\
  \texttt{RMSE} & [0, +∞) & 均方根误差。越小越好 \\
  \texttt{ubRMSE} & [0, +∞) & 去偏后的 RMSE = sqrt(RMSE² - bias²)。剔除均值偏差只看变化幅度 \\
  \texttt{CRMSD} & [0, +∞) & Centered RMSD（同 ubRMSE 概念） \\
  \texttt{mean\_absolute\_error} & [0, +∞) & 绝对偏差均值 \\
  \texttt{correlation} & [-1, 1] & Pearson 相关系数。1 = 完美正相关 \\
  \texttt{correlation\_R2} & [0, 1] & 相关系数平方，可解释方差比例 \\
  \texttt{NSE} & (-∞, 1] & Nash-Sutcliffe Efficiency。1 = 完美；0 = 等价于均值；< 0 = 比均值还差 \\
  \texttt{KGE} & (-∞, 1] & Kling-Gupta Efficiency，把 bias / 相关 / 方差比合并打分 \\
  \texttt{KGESS} & (-∞, 1] & KGE Skill Score（相对参照） \\
  \texttt{index\_agreement} & [0, 1] & Willmott index。1 = 完美 \\
  \texttt{kappa\_coeff} & [-1, 1] & 类别一致性（适合分类型变量） \\
  \texttt{rv} & [0, +∞) & 方差比 σ\_sim / σ\_obs。1 = 一致 \\
  \texttt{ubNSE/ubKGE} & ... & 去偏版本 \\
  \bottomrule
\end{longtable}

\openbench{} 内置 25+ metric。完整列表与公式见开发卷第 6 章。

\subsection{用 \yamlkey{metrics} 限定子集}

默认计算全部可用 metric（很多）。如果只关心几个：

\begin{minted}{yaml}
metrics:
  - bias
  - RMSE
  - correlation
  - KGE
\end{minted}

只计算这 4 个。\file{<sim>\_metrics.csv} 只含 4 列。

\section{Scores 解读}

Scores 把多个 metric 按归一化方式合并成 \emph{0-1 之间} 的可比分数（1 = 完美）：

\begin{itemize}
  \item \texttt{nBiasScore}：基于 bias 的归一化打分
  \item \texttt{nRMSEScore}：基于 RMSE
  \item \texttt{nPhaseScore}：相位一致性（季节循环对齐）
  \item \texttt{nIavScore}：interannual variability 一致性
  \item \texttt{nSpatialScore}：空间分布一致性
  \item \texttt{nSeasonalityScore}：季节性一致性
  \item \texttt{Overall\_Score}：综合分（加权平均）
\end{itemize}

CSV 格式：

\begin{minted}{text}
ref,nBiasScore,nRMSEScore,nPhaseScore,nIavScore,nSpatialScore,Overall_Score
GLEAM_v4.2a_LowRes,0.84,0.71,0.68,0.79,0.82,0.77
\end{minted}

\textbf{Overall\_Score} 是 portrait plot 与 ranking 的核心输入；多模型比较时，它是单一最直观的"模型 X 对该变量评估得了 0.77 分"。

\subsection{用 \yamlkey{scores} 限定子集}

\begin{minted}{yaml}
scores:
  - nBiasScore
  - nRMSEScore
  - Overall_Score
\end{minted}

\section{聚合权重：\yamlkey{weight}}

\yamlkey{project.weight} 控制 \openbench{} 把 grid 数据"压成一个数"时如何加权。它\emph{不}影响 \file{metrics/} 与 \file{scores/} 下逐格点 metric NC 本身——这些永远是逐格点计算。它\emph{只}影响下游聚合：每张 IGBP/PFT/气候带分类表里的"Overall"列、各 class 列。

\subsection{默认值的细微差别}

schema 字段类型是 \texttt{Optional[str]}，Python 默认 \texttt{None}。但 \modname{config.adapter:647} 在传给引擎前做了一次替换：

\begin{minted}{python}
"weight": cfg.project.weight or "area"
\end{minted}

也就是：

\begin{itemize}
  \item YAML 不写 / 写 \yamlkey{weight: null} / \yamlkey{weight:~} → cfg.project.weight=\texttt{None} → 引擎实际用 \texttt{"area"}（cos-lat 面积权）
  \item YAML 写 \yamlkey{weight: none} → cfg.project.weight=\texttt{"none"}（字符串）→ 引擎匹配 else 分支 → 等同算术平均
  \item YAML 写 \yamlkey{weight: area} 或 \yamlkey{weight: mass} → 直接使用对应权重
\end{itemize}

\textbf{注意这个反直觉}：\emph{省略字段}和\emph{显式写 \texttt{none}}得到的是\emph{两种不同的}加权行为。绝大多数用户不写这个字段，就在用 \texttt{area} 加权而不自知。这是 \openbench{} 一个有意的设计选择（"不写就用对的"），但写文档时容易搞错。

\subsection{三种取值}

\textbf{\texttt{none}（仅当显式写）}：简单算术平均。
\begin{minted}{python}
overall_mean = ds[score].mean(skipna=True)
\end{minted}
所有有效格点等权——南极的格点和赤道格点权重一样。最容易理解，但有偏差：高纬区域格点面积更小（cos $\phi$ 因子），算术平均高估了高纬区域的影响。

\textbf{\texttt{area}}：余弦纬度面积权。
\begin{minted}{python}
weights = np.cos(np.deg2rad(ds.lat))
overall_mean = ds[score].weighted(weights).mean(skipna=True)
\end{minted}
对应 grid cell 的真实面积。\textbf{这是大多数全球或区域评估的正确选择}——你想表达"全球平均"时，就该用面积权重。

\textbf{\texttt{mass}}：面积权 $\times$ 通量权。
\begin{minted}{python}
area_weights = np.cos(np.deg2rad(ds.lat))
flux_weights = np.abs(ref_data.mean("time"))
combined = area_weights * flux_weights
weights = combined / combined.sum()
overall_mean = ds[score].weighted(weights).mean(skipna=True)
\end{minted}
让"哪里通量大"决定权重——对 ET、GPP、降水这类\emph{通量}变量，把热带、降水带这种"高活跃区"的 score 加权放大。\openbench{} 用 reference 数据（不是 sim）计算 flux\_weights，避免不同 sim 引入不同权重。

\subsection{选择建议}

\begin{itemize}
  \item \textbf{开发期 / debug}：显式写 \yamlkey{weight: none}（算术平均最容易理解逐格点贡献）
  \item \textbf{全球或区域定量评估}：不写或写 \yamlkey{weight: area}（cos-lat 面积平均，是\emph{省略时的实际行为}）
  \item \textbf{通量类变量发表论文}：写 \yamlkey{weight: mass}（强调高通量区域）
  \item \textbf{站点评估}：weight 不生效（站点本就是离散点，无网格面积概念）
\end{itemize}

\subsection{陷阱：schema 注释和实际取值不一致}

代码 \modname{config/schema.py:31} 注释写 \texttt{area | equal}，但 \openbench{} 引擎 \modname{core/landcover_groupby.py} 实际识别的字符串是 \texttt{area} 与 \texttt{mass}。任何其他值（包括 \texttt{equal}）都走 \texttt{else} 分支等同于 \texttt{none}。这是 schema 注释还没跟上实现的轻微脱节，写配置时认实现，别认注释。

\section{按区域分类聚合（IGBP / PFT / 气候带 groupby）}

\openbench{} 内置三种"按地表/气候属性切片"的分类聚合，开关分别是 \yamlkey{project.IGBP_groupby} / \yamlkey{project.PFT_groupby} / \yamlkey{project.climate_zone_groupby}（都是 \texttt{bool}，默认 \texttt{false}）。它们共用一套机制：

\begin{enumerate}
  \item 加载内置静态分类图（\file{IGBP.nc} / \file{PFT.nc} / \file{Climate\_zone.nc}）
  \item 用 CDO \texttt{largest\_area\_fraction\_remap} 重网格到 \yamlkey{project.grid_res}
  \item 对每个 metric/score grid，按 class id 切片：\code{ds.where(IGBPtype == i)}
  \item 在每个切片内计算\textbf{中位数（metric）和（加权）平均（score）}
  \item 输出三件套：每 class 一个 NC + 一张总览 CSV + 一张 LC\_based\_heat\_map PNG
\end{enumerate}

\subsection{三种分类的差别}

\begin{center}
\begin{tabular}{|l|c|p{4cm}|p{4cm}|}
\hline
\textbf{分类} & \textbf{class 数} & \textbf{含义} & \textbf{何时用} \\
\hline
\texttt{IGBP\_groupby} & 17 & MODIS 地表覆盖 & 区分常绿林/草地/作物/城市/水体 \\
\hline
\texttt{PFT\_groupby}  & 16 & 植物功能型 & 模型 PFT 评估（CoLM、CLM 等） \\
\hline
\texttt{climate\_zone\_groupby} & 5--12 & K\"oppen 气候带 & 分热带/干旱/温带/极地评估 \\
\hline
\end{tabular}
\end{center}

\subsection{输出位置}

\file{<output\_dir>/<project\_name>/comparisons/<groupby\_name>/<sim>\_\_\_<ref>/}

下列产物（以 IGBP 为例）：

\begin{itemize}
  \item \file{<var>\_ref\_<R>\_sim\_<S>\_<metric>\_IGBP\_<className>.nc}：每 class 切片
  \item \file{<var>\_<sim>\_\_\_<ref>\_metrics.csv}：行=metric，列=每 class + Overall
  \item \file{<var>\_<sim>\_\_\_<ref>\_scores.csv}：行=score，列同上
  \item \file{<var>\_<groupby\_name>\_heatmap\_metrics.png}：可视化
\end{itemize}

\subsection{静态数据的查找路径}

三个分类文件按以下顺序查找（\modname{core.landcover_groupby._resolve_static_dataset}）：

\begin{enumerate}
  \item \env{OPENBENCH\_DATASET\_DIR}\code{/<filename>} 若环境变量已设
  \item \file{./dataset/<filename>}（cwd 相对，向后兼容老布局）
\end{enumerate}

\textbf{常见错误}：在容器或 SLURM 作业里跑而 cwd 不是项目根，又没设 \env{OPENBENCH\_DATASET\_DIR}，会得到 \texttt{FileNotFoundError: ./dataset/IGBP.nc}。建议在 \texttt{.env} 或 \cli{run} 前 \texttt{export} 一次。

\subsection{三个 groupby 都开还是只开一个}

\begin{itemize}
  \item 只开你真要看的那一个：每个 groupby 都触发一次 CDO 重网格 + 全 metric 全 class 切片，开三个会把 \file{comparisons/} 阶段时间拉到 3×
  \item 站点评估下：groupby 仍可工作（按站点所在 class 分组），但样本量更敏感——少 class 可能只有 1-2 个站，统计意义有限
  \item 与 \yamlkey{weight} 联动：groupby 输出 CSV 里的 \texttt{Overall} 那列就用 \yamlkey{weight} 指定的方法聚合（\texttt{none/area/mass}）
\end{itemize}

\section{Comparison 输出（多模型对比）}

\yamlkey{comparison.enabled: true} 时，\file{comparisons/} 下产出多个图种，每种一个子目录。每子目录里有：聚合 CSV（一行 = 一个 (item, ref, sim) 组合）+ 逐图 NC（每对一个）+ 最终 PNG。

\subsection{核心图种}

\textbf{Portrait\_Plot\_seasonal} \file{Portrait\_Plot\_seasonal/Portrait\_Plot\_seasonal.csv} 列：\texttt{Item} / \texttt{Reference} / \texttt{Simulation} + 每 metric/score 的 4 个季节分量（DJF / MAM / JJA / SON）。配套 PNG 把全部行作为热图：x 轴 = simulation × 季节；y 轴 = (item, ref, metric)；色阶 = 数值。

\textbf{HeatMap} \file{HeatMap/} 全 metric/score 跨 (item, sim, ref) 的二维热图汇总，按 weight 设置加权（详见本章 §"聚合权重"）。

\textbf{Single\_Model\_Performance\_Index (SMPI)} \file{Single\_Model\_Performance\_Index/} 每个 sim × ref 一个综合得分（基于均方差 / 观测方差），辅以 bootstrap 置信区间。这是\emph{ranking 的实际数据来源}。

\textbf{Taylor\_Diagram / Target\_Diagram} \file{Taylor\_Diagram/}、\file{Target\_Diagram/} 经典两种"标准化对比图"，把 (相关、归一化标准差) 或 (bias, ubRMSE) 二元投影到极坐标。每图叠合所有 (sim, ref) 点。

\textbf{Kernel\_Density\_Estimate / Ridgeline\_Plot / Whisker\_Plot} \file{Kernel\_Density\_Estimate/} 等 概率密度对比、脊线图、箱线图，用于看 sim/ref 分布形状差异。

\textbf{Parallel\_Coordinates / Diff\_Plot / Relative\_Score} \file{Parallel\_Coordinates/} 等 多 metric 联合视图、相对偏差地图、相对评分。

\textbf{Mann\_Kendall\_Trend\_Test} \file{Mann\_Kendall\_Trend\_Test/} 趋势显著性检验输出（统计 p 值、Sen's slope）。

\subsection{Groupby（仅条件触发）}

仅当 \yamlkey{IGBP_groupby} / \yamlkey{PFT_groupby} / \yamlkey{climate_zone_groupby} 任一为 \texttt{true} 时产出（详见本章 §"按区域分类聚合"）。每子目录按 \file{<sim>\_\_\_<ref>/} 切分，再按 metric/score 出 NC + 总览 CSV + LC heatmap PNG。

\subsection{多 ref 在 comparison 中的呈现}

\openbench{} v3.0 多 ref 配置下，所有 (sim, ref) 对都进入同一组 comparison 输出：

\begin{itemize}
  \item 聚合 CSV：每对一行，\texttt{Reference} 列区分（同变量 \texttt{Item} + 不同 \texttt{Reference} → 多行）
  \item Portrait/HeatMap PNG：把 \texttt{(item × ref)} 当作"扩展的变量"放在 y 轴，所以 2 ref × 5 metric = 10 行图块
  \item Taylor/Target/SMPI：每对单独投点，图例区分
  \item Groupby：按 \file{<sim>\_\_\_<ref>/} 拆目录，自然分开
\end{itemize}

\textbf{注意}：comparison 阶段并\emph{不}做"跨 ref 平均到一个数"——它把每个 ref 都展示出来，让你自己判断模型对 ref 选择的鲁棒性。如果你想要"跨 ref 平均"的最终评价，需要在 \file{Portrait\_Plot\_seasonal.csv} 上自己做后处理（pandas \texttt{groupby("Item")} 后 \texttt{mean()}）。

\section{Statistics 输出}

\yamlkey{statistics.enabled: true} 时，\file{statistics/} 下按统计模块名分目录：

\begin{itemize}
  \item \texttt{ANOVA/}：方差分解
  \item \texttt{Correlation/}：相关与协方差
  \item \texttt{Hellinger\_Distance/}：分布相似度
  \item \texttt{Mann\_Kendall\_Trend\_Test/}：趋势显著性
  \item \texttt{Kernel\_Density\_Estimate/}
  \item \texttt{Three\_Cornered\_Hat/}：三角帽误差分解
  \item \texttt{Partial\_Least\_Squares\_Regression/}
  \item \texttt{Functional\_Response/}
  \item ...
\end{itemize}

每个目录内含 CSV 数值结果 + PNG 可视化。

\subsection{用 \yamlkey{statistics.items} 限定}

\begin{minted}{yaml}
statistics:
  enabled: true
  items:
    - Correlation
    - Mann_Kendall_Trend_Test
\end{minted}

只跑这两个，省时。

\section{Visualization 产物（figures/）}

\file{figures/<var>/} 下按图种分子目录，每个 sim×ref 对一张：

\begin{itemize}
  \item \texttt{geo/}：地理分布图（mean / bias / correlation 三种 panel）
  \item \texttt{portrait/}：portrait sub-plot
  \item \texttt{radar/}：单变量雷达
  \item \texttt{correlation/}：散点图（sim vs ref）
  \item \texttt{kernel\_density\_estimate/}：KDE 概率密度对比
  \item \texttt{ridgeline/}：脊线图
  \item \texttt{heatmap/}：相关矩阵热图（多变量时）
  \item \texttt{LC\_based\_heat\_map/}：按 IGBP 分类的 heatmap（仅 \yamlkey{IGBP_groupby: true}）
  \item ...
\end{itemize}

\openbench{} 内置 17 个 \texttt{Fig\_*.py} 模块。开发卷第 7 章会介绍如何添加新图种。

\section{常见解读问题}

\subsection{为什么我的 metric 是 NaN}

\begin{itemize}
  \item Sim 与 ref 时间窗口无重叠 → check 应该已经报警
  \item 空间网格全为 NaN（落入海洋 / 极地）→ 检查 \yamlkey{lat_range}/\yamlkey{lon_range}
  \item 单位换算翻车（变量值变成无穷大）→ 看 \cli{openbench model show <model>} 的 compute 表达式
  \item NaN 比例超过 \yamlkey{min_year_threshold} 阈值 → 提示数据稀疏
\end{itemize}

\subsection{为什么 Overall\_Score 比单 metric 还低}

Overall\_Score 是多个 score 的平均。某个 score（如 spatial）很差会拉低均值。看分量 score：

\begin{minted}{bash}
cat output/<run>/scores/Evapotranspiration/MyModel_scores.csv
\end{minted}

\subsection{Comparison 图为空}

只有 1 个 simulation 时，comparison 图退化为单模型自比，往往看不出差异。多 simulation 才有意义。
