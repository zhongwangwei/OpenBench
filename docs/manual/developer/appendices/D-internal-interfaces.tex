\chapter{内部 interfaces}

本附录列出 \openbench{} 中的 ABC 与 Protocol 抽象类，由 \cli{make generated} 从 \file{src/openbench/\{util,core,data,runner\}/} 扫描产出。

这些类是 \openbench{} 的内部"插槽"：要替换某层实现（比如换缓存策略、换评估引擎）时，从这里着手实现新子类，而不是改主线代码。

\warninline{\textbf{这些是内部接口}：从外部脚本不要直接 import；签名可能在小版本间变化。如果你的扩展需要继承这些类，请先看附录 B 的"API 稳定性承诺"或在 GitHub Issue 讨论。}
